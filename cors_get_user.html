Notes

  • Same-origin lab test. Open this file via file:// or any non-supplier.holmesops.ai origin (e.g. http://localhost:8000/cors_get_user.html or a public host). It must not be served from the supplier portal itself, or the request is same-origin and the test is meaningless.
  • SameSite caveat. The supplier portal sets the access_token cookie with SameSite=Lax. Modern browsers therefore drop cookies on this cross-site fetch, and the request will be authenticated as anonymous (HTTP 401). If that happens, the credentialed-CORS path is mitigated by SameSite — but the underlying CORS misconfiguration (preflight reflection + * on actual response with credentials) still stands as a finding because (a) any future drift to SameSite=None immediately turns this PoC into account-takeover, and (b) it allows blind state-changing CSRF on routes that bypass SameSite (top-level navigations).
  • If SameSite=None is ever set on access_token, this PoC will read the response body — which includes app_metadata.mfa_secret and mfa_otpauth_url. The bug then escalates to remote MFA-secret exfiltration with one click, no malware required.
